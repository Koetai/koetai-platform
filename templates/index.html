What is Koetai?

Koetai lets you host RDF knowledge graphs and expose them as FAIR, standards-compliant SPARQL endpoints — without managing server infrastructure yourself. It is designed for ontologists, bioinformaticians and Linked Data practitioners who want to share semantic data quickly and reliably.

Each dataset you create gets its own named graph, a public SPARQL endpoint, shape schemas (ShEx / SHACL), a Mermaid diagram and a parameterized REST API. Data can be loaded from local files, GitHub/GitLab repositories, web download pages, or directly from tabular formats like CSV, Excel and Parquet.

Load RDF data
Upload files up to 500 MB (Turtle, N-Triples, RDF/XML, OWL, JSON-LD …), pull from GitHub/GitLab repositories, or scrape RDF links from any download page. Optional OWL RL reasoning materializes inferred triples before indexing.
Shapes & validation
Infer ShEx or SHACL schemas automatically with ShExer. Visualise the schema as a Mermaid class diagram with per-property coverage statistics. Validate with Apache Jena or RUDOF.
REST API
Turn any SPARQL query into a parameterized REST endpoint ({{param}} placeholders → URL query params). Returns JSON; interactive test form included.
New: upload tabular data directly

No RDF? No problem. Upload a Parquet, CSV, Excel or JSON file and map its columns to ontology terms using the built-in mapping wizard — powered by Morph-KGC. Predicates and classes are looked up live from the EBI Ontology Lookup Service as you type, so you can map columns to standard ontology terms (HPO, GO, SNOMED, schema.org, …) without leaving Koetai.

Parquet CSV / TSV Excel JSON → RDF via YARRRML + Morph-KGC → SPARQL endpoint
